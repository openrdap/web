OpenRDAP is a Registration Data Access Protocol (RDAP) client, written in Go.

This is currently under construction, notably it does not print out RDAP responses yet. Brave gophers can checkout the 'initial' branch on GitHub. Otherwise, try the demo page.

What works so far:

  • Project
    • Structured as a Go RDAP library, with a command line client which uses the library.
    • Website for the command line client.
    • Go library discoverable via github/godoc/others.
    • Go library potentially can be used to build RDAP servers (build Go value for response, encode to RDAP response), perhaps provide an example for that.
  • Bootstrapping
    • Bootstrap support implemented (code)
    • Bootstrap supports all types (asn/ipv4/ipv6/dns)
    • Defaults to IANA's service, can also bootstrap from custom URLs
    • Test bootstrap service @ https://test.rdap.net/rdap/, which includes experimental servers not listed in IANA's service yet
    • Experimental support for object tags
  • Bootstrap cache
    • Supports caching of {asn,ipv4,ipv6,dns,...}.json both in memory, or on disk ($HOME/.openrdap ...).
    • Supports custom cache timeout
  • Decoder
    • Can decode all the RDAP response objects into Go values
    • Supports & stores unknown fields, so you can print these out
    • Supports JCard
  • Client
    • Supports automatic query type detection (rdap as2856; rdap example.cz; rdap 192.0.2.0...)
    • Supports queries to URLs
    • Supports timeouts
    • Supports disabling SSL verification
    • Reasonably good verbose output
    • "Sandbox" mode to disable changing the cache dir (so it can be run on /demo)
    • Usage is currently something like: 
      // Client implements an RDAP client.
//
// This client executes RDAP requests, and returns the responses as Go values.
//
// Quick usage:
//   client := &rdap.Client{}
//   domain, err := client.QueryDomain("example.cz")
//
//   if err == nil {
//     fmt.Printf("Handle=%s Domain=%s\n", domain.Handle, domain.LDHName)
//   }
// The QueryDomain(), QueryAutnum(), and QueryIP() methods all provide full contact information, and timeout after 30s.
//
// Normal usage:
//   // Query example.cz.
//   req := &rdap.Request{
//     Type: rdap.DomainRequest,
//     Query: "example.cz",
//   }
//
//   client := &rdap.Client{}
//   resp, err := client.Do(req)
//
//   if domain, ok := resp.Response.(*rdap.Domain); ok {
//     fmt.Printf("Handle=%s Domain=%s\n", domain.Handle, domain.LDHName)
//   }
//
// Advanced usage:
//
// This demonstrates custom FetchRoles, a custom Context, a custom HTTP client,
// a custom Bootstrapper, and a custom timeout.
//   // Nameserver query on rdap.nic.cz.
//   server, _ := url.Parse("https://rdap.nic.cz")
//   req := &rdap.Request{
//     Type: rdap.NameserverRequest,
//     Query: "a.ns.nic.cz",
//     FetchRoles: []string{"all"},
//     Timeout: time.Second * 45, // Custom timeout.
//
//     Server: server,
//   }
//
//   req = req.WithContext(ctx) // Custom context (see https://blog.golang.org/context).
//
//   client := &rdap.Client{}
//   client.HTTP = &http.Client{} // Custom HTTP client.
//   client.Bootstrap = &bootstrap.Client{} // Custom bootstapper.
//
//   resp, err := client.Do(req)
//
//   if ns, ok := resp.Response.(*rdap.Nameserver); ok {
//     fmt.Printf("Handle=%s Domain=%s\n", ns.Handle, ns.LDHName)
//   }
  • Command line client
    • Runs upto downloading the RDAP response, doesn't print it yet.
    • See the docs/demo pages for progress/intended features.
  • Website
    • Written in Go (revel), so it can run live OpenRDAP demos.

What needs doing:

  • Encoder: Go value to RDAP response object encoder
  • Client: Additional fetches for contact information
  • Client: Needs to be robust against large (say 128MB+) responses.
  • Output: RDAP object text formatter (print RDAP responses out in a WHOIS, human readable type form)
